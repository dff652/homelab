# TODO / Roadmap

> 本文件用于记录后续功能规划，当前不默认启用，避免影响现有稳定功能。

## P0（高优先级）

- [ ] **NVMe 温度监控告警阈值**
  - 在监控模式中增加 `warn/critical` 阈值配置（默认关闭）。
  - 采样输出中标记告警等级，便于定位异常时段。

- [ ] **温度趋势可视化增强**
  - 在 `scripts/plot_nvme_temperature.py` 中绘制阈值线（Warning/Critical）。
  - 增加采样区间统计（最小/最大/平均/95 分位）。

## P1（中优先级）

- [ ] **多设备/多文件对比绘图**
  - 支持一次输入多个 CSV，在同图对比 `Composite/Sensor1/Sensor2`。
  - 支持按设备名自动分图，减少人工整理成本。

- [ ] **RAID 物理盘映射增强**
  - 支持可选调用 `storcli/perccli`，输出逻辑盘与物理盘槽位映射。
  - 明确区分“控制器视角”和“块设备视角”，避免误判。

- [ ] **测速结果持久化**
  - 把读写测速结果落盘到 `logs/`，便于长期追踪性能漂移。
  - 记录测试模式（只读/读写、是否清缓存）以保证可比性。

## P2（低优先级）

- [ ] **兼容性扩展**
  - Armbian/OpenWrt 的最小化依赖检查与功能降级策略（探测后提示）。
  - BusyBox 环境下的命令替代路径整理。

- [ ] **长期监控方案**
  - 可选导出 Prometheus 文本格式指标（离线场景优先）。
  - 提供 Grafana 看板模板示例（文档级，不绑定部署）。

## 说明

- 原则：默认保持“只读优先、显式确认写入/系统变更”的安全策略。
- 任何涉及写入磁盘、清缓存、安装/卸载依赖的操作，必须继续保留风险提示与二次确认。
